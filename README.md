# 지역사회보장계획 수립을 위한 GIS분석

읍면동 단위 코로플레스 맵(choropleth map) 웹서비스. 사회복지 정책담당자가 별도 GIS 소프트웨어 없이 브라우저에서 통계값 입력 → 색상 시각화 → 보고서 첨부 가능한 PNG 산출까지 일괄 수행하도록 설계됨.

본 서비스는 한국보건사회연구원에서 제공함.

---

## 1. 서비스 개요

본 서비스는 지역사회보장계획 수립 과정에서 빈번히 요구되는 지역간 격차 시각화 작업을 지원함. 별도 설치 없이 브라우저에서 실행 가능하며, 입력 데이터는 사용자 단말 내에서만 처리됨(서버 전송 없음).

□ 주요 기능
- 보기 모드 3종 (전국 시군구 / 시도 내 읍면동 / 시군구 내 읍면동)
- 데이터 입력 2종 (직접 입력 / 엑셀 업로드)
- 색상 팔레트 6종 + 분류 방식 3종 + 구분 단계 5종(3~7) → 총 90종 조합
- PNG 내보내기 (보고서 삽입용 클린 모드 + 화면 캡처 모드)
- 엑셀 양식 자동 생성 (지역코드·지역명 사전 기입)

□ 기술 환경
- 정적 페이지 기반 (서버 불필요)
- Chrome / Edge / Safari / Firefox 최신 버전 권장
- 모바일·태블릿 반응형 지원

## 3. 사용 방법 (4단계 워크플로)

화면 하단 사용 안내 바에 진행 단계가 항상 표시됨. 단계별 완료 시 체크 표시 자동 갱신.

### 단계 ① 보기 모드 선택
좌측 상단 보기 모드 라디오 버튼에서 1종 선택:

| 모드 | 표시 단위 | 데이터 행 수 | 활용 예시 |
|------|---------|-------------|----------|
| 전국 → 시군구 | 시군구 (255개) | 255행 | 전국 단위 격차 진단 |
| 시도 → 읍면동 | 시도 1개의 읍면동 | 시도별 약 200~600행 | 광역 시도 내부 분포 |
| 시군구 → 읍면동 | 시군구 1개의 읍면동 | 시군구별 약 5~50행 | 기초자치단체 단위 정밀 분석 |

→ "시도 → 읍면동" 또는 "시군구 → 읍면동" 선택 시 드롭다운 추가 노출. 시도 변경 시 시군구 자동 초기화.

### 단계 ② 데이터 입력
입력 패널 상단 탭에서 입력 방식 선택.

**(가) 직접 입력 탭**
- 현재 보기 모드의 지역 목록 자동 표시
- 각 행에 값 입력 → 즉시 지도 색상 갱신
- 미입력 지역 → 회색(#cccccc) 처리
- 소규모 데이터(수십 건 이내) 또는 데모용 권장

**(나) 엑셀 업로드 탭**
- 양식 다운로드 버튼 → 현재 보기 모드 기준 양식 xlsx 자동 생성
- 양식 작성 후 업로드 → 자동 매칭 + 결과 미리보기 표시
- 매칭 성공·실패 건수 노출
- 매칭 실패율 50% 초과 시 노란색 경고 박스 출현 → 보기 모드 또는 양식 재확인 권고
- 실무 주력 권장 (수백~수천 건 처리)

### 단계 ③ 색상 설정

□ 팔레트 (6종)
- **YlOrRd**: 흰→노랑→주황→빨강 (저→고 단방향, 발생률·결핍률)
- **Blues**: 흰→파랑 (저→고 단방향, 단조 증가 지표)
- **Greens**: 흰→녹색 (저→고 단방향, 긍정 지표)
- **RdYlGn**: 빨강↔녹색 (양방향 발산, 긍정·부정 양극)
- **Spectral**: 다색 발산 (대비 강조)
- **Viridis**: 색각이상 친화 (학술 보고서 권장)

□ 분류 방식 (3종)
- **등간격(equal interval)**: 최대-최소 값을 단계 수로 균등 분할 → 분포 왜곡에 민감
- **분위수(quantile)**: 각 구간별 동일 빈도 분할 → 시각적 균형 우수 (디폴트)
- **자연단절(Jenks natural breaks)**: 분산 최소화 알고리즘 → 통계적 최적 (simple-statistics 패키지)

□ 구분 단계
- 3~7단계 라디오 버튼 선택
- 권장: 분위수 5단계 (대다수 학술보고서 표준)

### 단계 ④ PNG 내보내기

헤더 우측 "⬇ PNG" 버튼 클릭.

□ 클린 모드 (체크박스 ON, 디폴트 권장)
- OSM 타일 배경 일시 숨김 → 폴리곤만 출력
- 흰 배경 적용
- 좌측 상단 자동 제목 박스 ("지역사회보장계획 수립을 위한 GIS분석" + 지역명 + 모드 + 날짜)
- 좌측 하단 워터마크 (한국보건사회연구원 제공 + 데이터 출처)
- → 학술보고서·정책자료 첨부에 적합

□ 클린 모드 OFF
- OSM 타일 포함 화면 그대로 캡처
- 정황 설명·발표자료에 적합

□ 해상도 선택: 1x / 2x(디폴트) / 3x
- 보고서 인쇄 시 2x 이상 권장

→ 파일명 자동: `choropleth_{모드}_{지역}_{YYYY-MM-DD}.png`

### 그 외 인터랙션
- 마우스 호버: 지역명 + 값 툴팁 노출
- 클릭: 상세 팝업
- 줌·팬 자유
- 헤더 "↺ 리셋": 모든 입력값·설정 초기화 (변경사항 있을 때만 confirm 다이얼로그)

## 4. 엑셀 양식 작성 가이드

### 양식 다운로드 → 작성 → 업로드 절차

**1) 양식 다운로드**
엑셀 업로드 탭에서 "양식 다운로드" 클릭 → 보기 모드별 양식 자동 생성. 다운로드 파일은 시트 2종 포함:
- **입력양식** 시트: 지역코드 + 시도 + 시군구 + (읍면동) + 값 컬럼 사전 기입
- **사용방법** 시트: 작성·저장·매칭·오류 4개 섹션 안내

**2) 작성 시 유의사항**
- 지역코드 컬럼은 **텍스트(@)** 형식 자동 지정됨 → 앞자리 0 손실 방지
- 값 컬럼만 입력하면 됨 (지역코드·지역명은 매칭 키이므로 건드리지 말 것)
- 미입력 행은 회색 처리됨
- 음수값 허용 (변화율·증감 지표 등)

**3) 저장 시 주의**
- xlsx 형식 유지 (xls 호환되나 xlsx 권장)
- "다른 이름으로 저장" 시 형식이 csv·텍스트로 바뀌지 않도록 주의

**4) 업로드 매칭 로직**
- 1차: adm_cd 8자리 정확 매칭
- 2차: 길이 4·7·9자리 → 자동 0 패딩 (4→5, 7→8, 9→10)
- 3차: 행안부 10자리(adm_cd2) → 통계청 8자리 자동 변환

**5) 오류 발생 시**
- "지역코드 컬럼 미발견" → 양식 재다운로드 후 재시도
- "매칭 실패율 50% 초과" → 보기 모드 일치 여부 확인 (시군구 양식을 시도·읍면동 모드에 업로드 등)
- 에러 박스 안 "입력값 초기화 후 재시도" 버튼 동봉

## 8. FAQ · 트러블슈팅

**Q1. 지도가 표시되지 않음.**
→ 인터넷 연결 확인. OSM 타일 차단 방화벽 환경일 경우 PNG 클린 모드만 사용 가능. 단일 HTML 임베디드 버전(`index_embedded.html`) 사용 권장.

**Q2. 엑셀 업로드 시 매칭이 0건임.**
→ 보기 모드와 양식 일치 여부 확인. 시군구 양식을 읍면동 모드에 업로드한 경우 발생. 양식 재다운로드 후 재시도.

**Q3. 지역코드 앞 0이 사라짐.**
→ 엑셀에서 셀 형식이 "일반"으로 바뀐 경우 발생. 양식 다운로드 후 지역코드 컬럼은 그대로 두고 값만 입력 권장. 자동 보정 로직(4→5, 7→8, 9→10 패딩) 적용되어 있으나 안전을 위해 형식 유지 권장.

**Q4. PNG 다운로드 시 OSM 타일이 누락됨.**
→ CORS 정책 차이로 발생. 클린 모드 ON으로 폴리곤만 출력하거나, 클린 모드 OFF에서 화면 캡처 시도.

**Q5. 모바일에서 패널이 보이지 않음.**
→ 좌측 상단 햄버거 버튼(≡) 클릭 → 패널 토글.

**Q6. 데이터 갱신은 어떻게?**
→ 행정경계는 vuski/admdongkor 저장소 갱신 시 수동 트리거 방식. 자동 cron 미적용. 갱신 필요 시 PLAN.md 참조.

**Q7. 음수값 처리는?**
→ 음수 허용. 변화율·증감 지표 등 발산형 데이터는 RdYlGn·Spectral 팔레트 권장.

**Q8. 인쇄 시 색상이 다르게 보임.**
→ 모니터 sRGB와 인쇄 CMYK 차이. PNG export 후 인쇄 전 색상 보정 권장.
