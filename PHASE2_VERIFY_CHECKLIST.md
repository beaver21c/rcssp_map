## 검증 항목

### □ 1. 초기 화면 (전국 시군구 모드 기본값)
- [ ] 헤더 "전국 읍면동 코로플레스 맵" 표시
- [ ] 좌측 패널 폭 320px, 보기 모드 라디오 3종 노출
- [ ] 첫 라디오 "전국 → 시군구 비교" 선택 상태
- [ ] 지도 영역 우측 표시, OpenStreetMap 타일 옅게 깔림 (opacity 0.35)
- [ ] 시군구 폴리곤 255개 자동 표시 (회색 채움 #e2e8f0, 흰색 경계선)
- [ ] fitBounds 자동 줌 → 한반도 전체 화면 적합
- [ ] 하단 사용 안내 바 5단계 표시, 1단계 "지역 선택" 강조

### □ 2. 호버 인터랙션
- [ ] 임의 시군구 폴리곤 마우스 오버
- [ ] 폴리곤 fillOpacity 0.92 + 경계선 1.5px 검정 강조
- [ ] 툴팁 표시: "종로구 / 11010" 형태
- [ ] 마우스 아웃 시 원상복구

### □ 3. 보기 모드 전환 ② "시도 선택 → 읍면동"
- [ ] 라디오 클릭 → 모드 전환
- [ ] 좌측 패널에 시도 드롭다운 출현 (17개 옵션)
- [ ] 지도 빈 화면 + "좌측에서 시도를 선택하시오" 안내
- [ ] 드롭다운 "서울특별시" 선택
- [ ] 서울 emd 427개 폴리곤 로딩 (1초 이내)
- [ ] fitBounds 자동 줌 → 서울 영역 적합
- [ ] 호버 시 읍면동명 툴팁 (예: "사직동 / 11010530")

### □ 4. 보기 모드 전환 ③ "시군구 선택 → 읍면동"
- [ ] 라디오 클릭 → 모드 전환
- [ ] 시도 드롭다운만 노출 (시군구 드롭다운 미출현)
- [ ] "서울특별시" 선택 → 시군구 드롭다운 추가 출현
- [ ] 시군구 드롭다운 "종로구" 선택
- [ ] 종로구 17개 읍면동만 표시 (필터링 동작)
- [ ] fitBounds 자동 줌 → 종로구 영역 적합

### □ 5. 캐싱 검증
- [ ] 시도 모드에서 서울 → 부산 → 서울 재선택
- [ ] 두 번째 서울 선택 시 로딩 인디케이터 1초 미만 (캐시 적중)

### □ 6. 푸터·안내 바
- [ ] 푸터에 "vuski/admdongkor" 링크 정상 (새창 열림)
- [ ] 푸터에 "기준일 2026-04-01 · 변환 2026-04-28" 표시
- [ ] 사용 안내 바 1단계 "지역 선택" 완료 시 체크표시·연두색 변환 (값 입력 전이므로 2단계 강조)

### □ 7. 콘솔 에러 검증
- [ ] 브라우저 개발자도구(F12) 콘솔 → 에러 0건
- [ ] Network 탭 → TopoJSON·JSON 응답 모두 200 OK

### □ 8. 네트워크 성능
- [ ] sgg.topojson 응답 < 1초
- [ ] 시도별 emd 응답 < 1초 (경기 31번 가장 큼 208KB)
